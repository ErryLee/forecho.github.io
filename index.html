读书笔记

《需求》

个人认为这是一本产品经理必读的书之一。这本书本身围绕着 - 「只有真正帮用户解决麻烦地图的产品，才是好的产品」。

让顾客无可救药的爱上你的产品

把所有的时间和精力都投入到对「人」的了解上。创造出令人无法抗拒，更令竞争对手无法复制。【说实话，很难】

本书结合以下几个成功的产品来讲述的，让我来总结一下：

奈飞（Netflix）- 电影租凭

麻烦地图

当时的电影租凭让人很不愉快（迟到金）。

解决

没有罚金，也没有罚金引来的恼火，借还还非常方便。

诺基亚1100

令人影响深刻的科技产品

科技产品 销售记录
任天堂 Wii 游戏系统 4500万
摩托罗拉的 RAZR 系列手机 5000万
PlayStation 2 游戏机 1.25亿台
iPod 1.74亿台
诺基亚1100 （前5年） 2.5亿台

成功关键（靠洞察力和创造力取胜）

诺基亚1100 成功的关键在于设计。某些功能对生活在南亚、拉丁美洲和撒哈拉以南非洲农村地区的人们来说很有价值，其他一些功能则被简化或直接去除。同时，诺基亚1100 提供的一些可选功能，在西方人看来简直不可想象，例如，可以存储多套联系人名单，这是全村很多用户共用一部手机时不可或缺的功能。诺基亚1100 还允许用户为某次通话设置话费限额，这与在加油站预付费一个道理，也是支持手机公用的一个特性。此外，内置手电、收音机、闹钟等这些小功能对于居住在电力供应不稳定地区的人来说也大有用处。而且，诺基亚1100 的屏幕可以显示80 多种语言，还有帮助不识字用户进行辨识的图形标志。

Zipcar - 解决汽车共享服务

发现需求点

Zipcar会员常常会发现，在需求量最高的晚间和周末，这个城市中仅有的几辆车早就被别人预订一空。而且，最近的车辆也停在离家10~15个街区远。这虽然听起来不算什么大麻烦，但在整个Zipcar租车过程中再附加半个小时的步行时间，就足以让大多数客户望而却步。

与买车相比，Zipcar会员每年能节省几千美元的开支，而且还不用将大把的时间投入停车、维护、修理、保险等琐事之中。然而，真正促成人们加入Zipcar会员的原因，却是因为他们发现，车就停在离家步行5分钟的位置，而不是10分钟。如此看来，5分钟的差异才是触动需求的真正力量，这甚至比金钱和时间的节约对人们的影响力更大。

解决之道

先集中几个主要城市，密集提供租车，让他们5分钟之内租到车。让租车变的非常简单。

魏格曼商店（Wegmans） - 连锁商店

魔力

品类只有想不到，没有买不到的。更加干净。更加友好。商店的营业面积大得超乎你的想象。收银非常快，非常便捷。

工具控

我的 Chrome 扩展集

遇见 Chrome 浏览器

在大一的时候才开始拥有一台属于自己的电脑，那个时候买的还是二手台式机。之前的我可以说算是电脑小白了，根本就没有浏览器这个概念，就跟现在绝大多数的中国网民一样。只知道有个图标你点一下就能上网，后来才知道它就是 IE 浏览器。

也不知道什么时候开始什么原因，让我知道了 Chrome 浏览器，初次使用的时候，非常的不习惯。不习惯以下几点：

  1. 以标签形式展现一个网页。
  2. 头部很短。
  3. 界面极简。（但功能强大）

其实用了你才发现这些都是它的优点，再加上它的一个帐号多平台同步功能和扩展机制，我再也离不开它了。可以说 Chrome 浏览器把 「Don’t Make Me Think」发挥的淋漓尽致，这一点都不为过。

国内的什么猎豹浏览器，什么360极速浏览器都是在用 Chromium （Chrome 浏览器背后的引擎）基础上加一个「外壳」而已，但是就是用不习惯，用户体验太差。

「普遍」的扩展

这里是指使用率很高的扩展。

AdBlock

这个是用来屏蔽广告的，必备扩展，妈妈再也不用担心突然进入某个页面突然广告乱飞的情况了。再也不用担心使用百度时看到恶心的 flash 和大图广告了。

PS：有一个值得注意的情况：如果你是一个开发者，也就是工程师在写代码开发页面的时候，有可能启用这个扩展导致一些因屏蔽广告而导致的一些错位什么的问题，留一个心眼就好了。

Proxy SwitchySharp

翻墙必备扩展，翻墙教程我这儿有一个最简单的方法，屡试不爽。

豆藤 Bean vine

如果你经常使用豆瓣的话，用了这个扩展之后你就会停不下了的。

这个博客不是我的一个博客，之前用了3年的 WordPress 了，这个博客也刚开始用，目前只是为了体验以 Markdown 写作为主。

那么这个博客用来写什么呢？

如你所见，这个博客分两大类 - 技术栈和人文栈。

使用 Hexo 搭建一个免费的博客

我为什么要使用 Hexo 搭建一个博客？

之前一直在使用 WordPress 和 VPS 搭建了一个博客，并且还抽时间自己写了一个 WordPress 主题，作为自用。博客一直用到现在，用了3年多了，虽然写的不多，质量也不算高。但是已经养成写博客的习惯了。我相信这个习惯会让我受用一生。

但是 VPS 不稳定（便宜货，不靠谱，被坑过），所以我有装备份 WordPress 的插件，慢慢觉得也很麻烦。后来接触了 Markdown，作为一个标记语言，很好用，简洁。发现自己平常写文章的时候用的格式也不多，写文章的时候不能忍受为了给某个文字加粗，而去动用鼠标去点编辑器加粗的图标，这样显得很不优雅，也没有效率。而作为标记语言的 Markdown 缺很轻松的实现文本格式。

WordPress 文章存放在数据库中，感觉很不方便。对博客而言，使用文档存放文章，方便随时本地查阅是再好不过了。对，就是可以离线本地查阅，非常的好。就速度方面，使用数据库的 WordPress 显然没有直接读取静态文件的快。

综上所述，我决定去尝试其他可代替 WordPress 同时又能解决我上述顾虑的产品。

然后经好基友介绍我遇到了Pico，这个项目是一个 PHP 项目，这就意味着环境部署方面非常的方便。并且此项目最大的特点是直接读取 Markdown 文档，访问页面的时候把 Markdown 生成 HTML 格式的，此项目我折腾了两天，包括搭建看文档，自己移植一个主题过来，眼看一个文章详细页面就用移植完成的时候，发现我写的一个代码块直接给我在最外层加了一个 code 标签而已了，结果就是这个函数变成了一行，身为处女座，这事绝对不能让它发生，于是我放弃了此项目，连解决办法都没去找。

然后我就遇到了 Hexo，很早之前有听过这个项目，但是没搭建过。只是在 GitHub 搭建过一次 Jekyll，然后就没有然后了。Hexo 也满足我之前的要求，并且还可以结合 GitHub Pages 作为免费空间使用。刚好看到这个教程教你如何搭建 Hexo - 如何搭建一个独立博客——简明Github Pages与Hexo教程，于是，OK 就是它了。
